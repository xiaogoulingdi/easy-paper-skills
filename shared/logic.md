# 跨模块共享协议解析 (Shared Logic)

本文件是对 `easy-paper/shared/` 目录下文件的梳理说明。这里存放的是全局级别（Global）的共享架构设置。

由于本项目按阶段分为了三大主模块（深研 `deep-research`、写作 `academic-paper`、评审 `academic-paper-reviewer`），为了确保在不同模块切换时 AI 的设定不发生断裂或错乱，这里设定了三项核心的“流通公约”。

## 文件说明

### 1. `handoff_schemas.md` (交接数据结构标准)
**定义了不同模块之间“交接班”的数据规范。**
例如：当你用 `deep-research` 梳理完了文献，想要交给 `academic-paper` 里的 `draft_writer_agent` 开始写正文时，如何确保写作 Agent 能完全看懂研究 Agent 的结论？
这个文件使用结构化的表单/JSON格式，规定了“交接清单”该怎么写，防止上下文传递时的关键信息丢失。

### 2. `style_calibration_protocol.md` (学术风格校准协议)
**定义了全项目统一的学术口吻。**
不管你是雇佣了“摘要专家”还是“大纲架构师”，这个协议都会强制所有的智能体保持一种严谨、客观、学术的语气。它甚至提供了一套“从你的既往文章中学习你的写作口吻”的方法，防止写出来的东西充满浓浓的“机器AI味（比如大量使用‘首先、其次、总之’或者华丽且空洞的辞藻）”。

### 3. `cross_model_verification.md` (跨模型双重验证协议)
**用于防止 AI 幻觉和逻辑自圆其说。**
高级校验设置：当系统进行极度复杂的方法论推导，或者自我纠错时，这个协议规范了如何利用两套不同的子模型（例如先让模型 A 生成论据，再强制要求模型 B 充当第二双眼睛进行独立交叉审查），从而实现最高等级的逻辑闭环。
